Claude for Humanities Scholars

Part of A Hitchhiker's Guide to AI for the Humanities
✓ Last verified: March 2026

This is the operational manual for using Claude in humanities research and teaching. It covers setup, features, limits, and worked examples. It is not a general introduction to AI — for that, start with the Primer. For cross-platform principles like verification, data governance, and prompting, see Essentials.

👍 Don't Panic
If you only use Claude's conversational interface, read this overview and the Chat page. You can safely ignore Code, Cowork, hooks, MCP, CLAUDE.md, and everything in Environments and Prerequisites until you have a specific need.

Which Claude tool do I need?

Anthropic offers Claude through several interfaces. They are not competing products but points on a spectrum from conversational to agentic. Before going deep on any one tool, this guide will help you choose where to start.

ToolBest forYou'll need
ChatQuestions, writing help, document analysisFree account or Pro
ProjectsPersistent research contextAny plan (limits vary)
CoworkAutonomous file tasks, document creationPro or Max + Desktop app
CodeBuilding tools, data processing at scalePro, Max, or API key
✓ Minimum safe practice
  1. Use Chat. Upload the source text you want to work with.
  2. Ask for critique, summary, or analysis — not for authoritative facts.
  3. Verify all references against a library catalogue.
  4. Do not upload sensitive, personal, or embargoed material.
  5. Record what you asked and what you used from the response.
  6. If it's useful, great. If not, nothing was lost.

An important framing

The output of Claude Code need not depend on LLMs at all. It can produce entirely "traditional" digital outputs — static web pages, Python scripts, cleaned datasets, formatted bibliographies, visualisations — that work independently of any AI system. Claude Code is the builder; the thing it builds can be conventional software. This is a strength: you get AI assistance in the construction process without creating AI dependency in the result.

✏️ Leif's Notes
I've found that the single most valuable thing about Claude Code for my own DH work isn't the coding — it's the ability to iterate rapidly on ideas that would otherwise take weeks to prototype. The risk is that speed encourages sloppiness. My rule of thumb: if I can't explain to a colleague what the tool built and why, I haven't understood it well enough to use it in published work. Your mileage may vary, but I'd suggest starting with tasks where getting it wrong has low consequences.
🗺️ Off the Beaten Track
If you already have some programming experience, Claude Code can dramatically accelerate tool-building for DH projects. See Code and Worked Examples for what's possible — but read Prerequisites first to understand what foundational knowledge still matters.
⚠️ Caution
Claude's knowledge has a cutoff date and its outputs can contain plausible-sounding errors. Always verify factual claims, references, and dates against primary or authoritative sources. See Verification & Citation in Essentials.

The Primer

A practical introduction to AI for humanities and social sciences

This guide aims to help you make informed decisions about whether and how to engage with Large Language Models. It is not advocacy for adoption. Some colleagues will find these tools useful for specific tasks; others might reasonably conclude they are not worth the trade-offs. Both positions are defensible.

👍 Don't Panic
If you feel apprehensive about AI, you are in good company. Many colleagues share that feeling. This guide assumes no prior experience and no particular enthusiasm. The first steps are genuinely easy — a 15-minute exercise you can try right now — and the AI itself will help you learn how to use it.

Essentials

Cross-platform principles for working with AI responsibly

This section covers the practical and ethical principles that apply regardless of which AI platform you use. Think of it as the health & safety, money, and customs information you need before visiting any region.

If you use Claude, ChatGPT, Gemini, or any other tool, this section is for you. The platform manuals will link here rather than duplicating this material.

✓ Essential
At minimum, read Verification & Citation and Data Governance before using any AI tool for academic work. Everything else can wait until you need it.

AI Bootcamp

Ten structured experiments for humanities scholars · 2–4 hours each

These exercises are designed to build practical competence through doing. They increase in complexity and technical ambition, but the first five require no coding at all. Each includes a brief, a worked example, and reflection prompts.

🗺️ Off the Beaten Track
Exercises 6–10 involve more technical work: data cleaning, building small tools, and using agentic features. They're optional, but if you've enjoyed the first five, they'll show you what's possible with a bit more investment.
